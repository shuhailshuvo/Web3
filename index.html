Web 1.0 (Read Only) – 1989

Web 1.0 refers to the first stage of the World Wide Web evolution. Earlier, there were only a few content creators in Web 1.0 with a huge majority of users who are consumers of content. Personal web pages were common, consisting mainly of static pages hosted on ISP-run web servers, or on free web hosting services.
Web 2.0 (Read/Write) – 1999

Web 2.0 refers to worldwide websites which highlight user-generated content, usability, and interoperability for end users. Interaction and collaboration with each other are allowed by Web 2.0 in a social media dialogue as the creator of user-generated content in a virtual community.
Web 3.0 Semantic Web concept from 1999 & 2001

Web 3.0 is built upon the core concepts of
  • Semantic Web
  • Artificial intelligence (AI)
  • Machine learning (ML)
  • virtual assistants
  • Connectivity and ubiquity (IoT)
  • Decentralization (Trustless and permissionless)

What Next? Web 3.0 or Web3

Gavin Wood’s Web 3

The term Web3 was coined in 2014 by Ethereum co-founder Gavin Wood, and the idea gained interest in 2021 from cryptocurrency enthusiasts, large technology companies, and venture capital firms. Web3 is distinct from Tim Berners-Lee's 1999 concept for a semantic web, which has also been called "Web 3.0". Some people referring to the decentralized concept usually known as "Web3" have used the terminology "Web 3.0", leading to some confusion between the two concepts. Furthermore, some visions of Web3 also incorporate ideas relating to the semantic web.

How to build dApps


Web3 SDK

Web3 has powerful SDKs and tools that make development a breeze - especially if you know Web2. If you’re familiar with frontend or backend libraries like React or NodeJS, you’re halfway there.

Metamask

In Web2, we primarily use email and passwords; these are tied in some way to our identity. In Web3, we can login using a wallet like MetaMask These wallets don’t require any form of personal ID. Instead, they use public and private key cryptography.

MetaMask is a popular software cryptocurrency wallet that is used to interact with the Ethereum Blockchain. A software cryptocurrency wallet allows users to hold and control their digital assets through a browser extension or mobile app. It is also used to authenticate and interact with Web3 applications.

Ethereum Boilerplate

There are a variety of boilerplates and starter projects to help you build Web3 dApps fast. The most popular is the Ethereum React Boilerplate which you can find here: https://github.com/ethereum-boilerplate/ethereum-boilerplate This boilerplate will save you months of work, as it comes with the most important Web3 hooks and components built-in.

Using Moralis in to connect your dApp to the blockchain speeds up dApp development by 80% or more. All the data you need is already extracted and indexed from the blockchain for you.
